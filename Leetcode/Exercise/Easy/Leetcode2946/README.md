# LeetCode 2946. Matrix Similarity After Cyclic Shifts

## 1. 核心思維：循環移位與同餘 (Modulo)

我們的目標是驗證將矩陣 `mat` 的奇數列向右循環移位 `k` 步、偶數列向左循環移位 `k` 步後，是否與原本的矩陣完全相同。

與其建立一個新的矩陣來模擬移位過程，我們可以利用數學上的「同餘 (Modulo)」去推算每個位置上的元素移位後應該對應到哪裡，然後直接在原矩陣上進行比對。

## 2. 解題策略：座標映射與就地檢查

對於矩陣中的任意元素 `mat[i][j]`（其中 `m` 是列數，`n` 是行數）：

1. **奇數列** (`i & 1` 為真)：
   - 元素向右循環移位 `k` 步。
   - 原始在索引 `j` 的元素將會移到索引 `(j + k) % n`。
   - 這意味著移位後的矩陣在 `(j + k) % n` 的值必須等於原矩陣在 `j` 的值。我們直接檢查 `mat[i][(j + k) % n] == mat[i][j]` 是否成立。

2. **偶數列** (`i & 1` 為假)：
   - 元素向左循環移位 `k` 步。
   - 原始在索引 `j` 的元素將會移到索引 `j - k` 的位置。
   - 為了處理 C++ 負數取餘的狀況，先計算出左移後的索引 `after_index = j - k`，若數值小於 0，則不斷加上 `n` 直至為正數為止。
   - 接著檢查 `mat[i][after_index % n] == mat[i][j]` 是否成立。

只要發現任何一個位置的對應元素不相等，即可提早結束並回傳 `false` (`check = false; break;`)。若遍歷整個矩陣後皆相等，則回傳 `true`。

## 3. 複雜度分析

- **時間複雜度**：$O(m \times n)$。其中 $m$ 和 $n$ 分別為矩陣的列數與行數。我們需要雙層迴圈遍歷矩陣中的每一個元素來進行常數時間的判斷。
- **空間複雜度**：$O(1)$。整個檢查過程僅使用到幾個整數型態的變數作為迴圈計數器與狀態標記，是在原矩陣上就地比對，不需配置額外的記憶體空間來複製矩陣。
