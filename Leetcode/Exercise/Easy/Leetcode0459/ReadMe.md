# LeetCode 459. Repeated Substring Pattern

## 1. 核心思維：KMP 演算法 (KMP Algorithm) 中 Next 陣列的應用

我們的目標是判斷一個字串 `s` 是否可以由它的某個子字串重複多次構成。

這個問題可以透過 KMP 字串匹配演算法中的 Next 陣列（也常稱為 `pi` 陣列或前綴函數）來優雅地解決。Next 陣列記錄了字串中每個前綴的最長相同真前後綴長度。

## 2. 解題策略：Next 陣列的性質

### 為什麼可以使用 Next 陣列？

Next 陣列 `pi[i]` 代表長度為 `i+1` 的前綴 `s[0...i]` 中，最長相同真前後綴的長度。
對於長度為 `m` 的字串 `s`，其最長相同真前後綴的長度即為 `pi[m-1]`。

一個重要的數學性質是：如果字串 `s` 是由某個長度為 `L` 的子字串重複多次構成，那麼：
1. `s` 必定具有相同的前綴和後綴，所以 `pi[m-1]` 必定大於 `0`。
2. 最長相同真前後綴的長度會剛好等於總長度減去一個重複子字串的長度，即 `pi[m-1] = m - L`。
3. 也就是說，最小可能重複的子字串長度 `L = m - pi[m-1]`，且字串的總長度 `m` 必定能被 `L` 整除（即 `m % (m - pi[m-1]) == 0`）。

### 演算法步驟

1. **計算 Next 陣列**：
   - 宣告一個長度為 `m` 的陣列 `pi`，並將 `pi[0]` 設為 `0`。
   - 使用變數 `k` 來記錄當前最長相同真前後綴的長度（即 `s[0...k-1]` 與 `s[q-k...q-1]` 匹配）。
   - 對於 `q` 從 `1` 到 `m-1` 遍歷：
     - 如果遇到 `s[k] != s[q]`，表示無法繼續延伸，必須讓 `k` 往前回退，利用已經計算好的 `pi` 陣列：`k = pi[k-1]`，直到 `k == 0` 或者找到 `s[k] == s[q]`。
     - 如果 `s[k] == s[q]`，表示前後綴可以延伸，此時 `k` 增加 `1`。
     - 儲存當前 `q` 位置的最長相同真前後綴長度：`pi[q] = k`。

2. **驗證重複子字串性質**：
   - 檢查 `pi[m-1] != 0`：若為 0，代表整個字串沒有相同的真前後綴，故無法由子字串重複構成。
   - 檢查 `m % (m - pi[m-1]) == 0`：若條件成立，則證實字串是由長度為 `m - pi[m-1]` 的子字串重複構成。
   - 符合上述兩個條件回傳 `true`，否則回傳 `false`。

## 3. 複雜度分析

- **時間複雜度**：$O(m)$，其中 $m$ 為字串 `s` 的長度。計算 Next 陣列時，儘管內部有一個 `while` 迴圈控制回退，但 `k` 的增加總次數不超過 $m$，每次進入 `while` 迴圈都會減少 `k`，減少的總次數也被限制在增加的總次數之內，因此均攤下來的時間複雜度為 $O(1)$，整體的迴圈遍歷時間為 $O(m)$。
- **空間複雜度**：$O(m)$，我們需要一個長度與字串相同的陣列 `pi` 來儲存過程中的最長相同真前後綴長度。
