# LeetCode 15. 3Sum

## 1. 核心思維：降維與雙指標

我們的目標是在陣列中找到所有和為 0 的不重複三元組 `(nums[i], nums[j], nums[k])`。

直接使用三層迴圈尋找會導致 $O(N^3)$ 的時間複雜度，這在多數情況下會超時。因此，我們可以透過 **排序 (Sorting)** 將問題簡化，並利用 **固定一個數字，再用雙指標 (Two Pointers) 找另外兩個數字** 的策略，將時間複雜度大幅降至 $O(N^2)$。

## 2. 解題策略：排序 + 雙指標

### 為什麼需要先排序？

1. **方便使用雙指標**：排序後的陣列具有單調性，可以根據當前三數之和與目標值 (0) 的大小關係，明確決定是要往右移動左指標（使和變大），還是往左移動右指標（使和變小）。
2. **容易略過重複元素**：為了確保結果中的三元組不重複，排序後相同的數字必定會相鄰。我們可以輕鬆利用條件 `nums[i] == nums[i-1]` 以及雙指標移動時的檢查，來略過已處理過的數字。
3. **提前結束迴圈**：如果固定的第一個數字已經大於 0，因為陣列已由小到大排序，後面的數字必然也都大於 0，三數之和絕對不可能為 0，此時可以直接中斷迴圈。

### 雙指標尋找機制

令當前固定的數字位置為 `i`。將左指標設為 `left = i + 1`，右指標設為 `right = n - 1`。
計算當前的三數之和 `sum = nums[i] + nums[left] + nums[right]`：

- 若 `sum == 0`：找到一組合法解，加入答案陣列中。接著左右指標繼續向內收縮，並**略過所有重複的數字**以避免產生重複解。
- 若 `sum < 0`：表示總和太小，我們需要較大的數字，因此將左指標向右移 (`left++`)。
- 若 `sum > 0`：表示總和太大，我們需要較小的數字，因此將右指標向左移 (`right--`)。

## 3. 演算法步驟

1. **排序陣列**：對原陣列 `nums` 進行從小到大排序。
2. **遍歷第一個數字**：使用迴圈變數 `i` 從 `0` 遍歷到 `n-2`。
   - **提早結束**：若 `nums[i] > 0`，代表後續和不可能為 0，直接 `break` 結束外層迴圈。
   - **去重檢查**：若 `i > 0` 且 `nums[i] == nums[i-1]`，代表這個數字已經作為基準探討過，加上 `continue` 略過。
3. **雙指標收縮**：
   - 初始化 `left = i + 1`, `right = n - 1`。
   - 進入 `while (left < right)` 迴圈：
     - 計算當前三數總和 `sum`。
     - 如果 `sum == 0`：記錄答案 `{nums[i], nums[left], nums[right]}`，同時讓 `left` 和 `right` 一路略過相同的數字，直到遇到不同的數值，最後 `left++, right--`。
     - 如果 `sum < 0`：`left++`。
     - 如果 `sum > 0`：`right--`。
4. **回傳結果**：回傳記錄的所有不重複三元組 `ans`。

### 複雜度分析

- **時間複雜度**：$O(N^2)$。排序陣列需要 $O(N \log N)$ 時間，外層迴圈遍歷約 $N$ 次，內層雙指標掃描每次最多耗費 $O(N)$，總體主導的時間複雜度為 $O(N^2)$。
- **空間複雜度**：$O(1)$ 或 $O(\log N)$。取決於語言內建排序演算法的實作細節（不計入儲存最終答案結果的矩陣空間）。
