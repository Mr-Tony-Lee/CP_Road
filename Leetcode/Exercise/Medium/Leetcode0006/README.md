# LeetCode 6. Zigzag Conversion

## 1. 核心思維：模擬法 (Simulation)

我們的目標是將給定的字串 `s` 根據指定的行數 `numRows` 進行「Z」字型（Zigzag）排列，然後按列讀取字元組成新的字串。

直觀的作法是直接模擬字元放入每一行的過程。我們可以使用多個字串來代表每一行，接著依次遍歷字串 `s` 中的每個字元，將它加到正確的行中。由於擺放順序是先往下到底，再往上到頂，不斷來回折返，我們可以利用一個變數去控制當前行數移動的方向。

## 2. 解題策略：陣列紀錄各行字串與方向控制

我們可以利用一個長度為 `numRows` 的字串陣列（`vector<string>`）`row_s` 來紀錄每一行的內容。

1. **處理邊界情況**：
   - 當給定的行數 `numRows == 1` 或是 `numRows >= s.size()` 時，排出來的形狀不會有折返的效果，結果與原字串相同，因此直接回傳 `s` 即可。

2. **初始化**：
   - 初始化行索引 `now_row = 0`，代表從第一行開始。
   - 初始化布林變數 `dir_down = true`，用來標示當前的移動方向是否為「向下」。

3. **模擬與附加字元**：
   - 用迴圈遍歷字串 `s` 的每個字元。
   - 將當前字元附加到對應的行陣列 `row_s[now_row]` 後面。
   - 準備移動到下一行：根據方向變數 `dir_down`，若向下則 `now_row + 1`，若向上則 `now_row - 1`。
   - 檢查更新後的行索引是否碰到邊界（第一行 `0` 或最後一行 `numRows - 1`）：
     - 如果是，則將方向反轉（`dir_down = !dir_down`），確保下一個字元能變換方向。

4. **組合最終字串**：
   - 遍歷完成後，將 `row_s` 中所有列的字串依序往下串接在一起，回傳最終的字串結果。

## 3. 複雜度分析

- **時間複雜度**：$O(n)$。其中 $n$ 為字串 `s` 的長度。遍歷字串把字元分配到各行需花費 $O(n)$，最後組合所有字串也花費 $O(n)$ 的時間，整體操作次數與字串長度成正比。
- **空間複雜度**：$O(n)$。需要額外的陣列 `row_s` 來儲存總共 $n$ 個字元（分散在各行中），組合最終回傳的字串也同樣需要 $O(n)$ 的空間，故整體空間複雜度為 $O(n)$。
