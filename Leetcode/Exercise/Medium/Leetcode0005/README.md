# LeetCode 5. Longest Palindromic Substring

## 1. 核心思維：中心擴展法 (Expand Around Center)

我們的目標是找出給定字串 `s` 中，最長的回文子字串 (Palindromic Substring)。

直觀的作法是檢查所有可能的子字串是否為回文，但這會產生 $O(n^3)$ 的時間複雜度，效率太低。我們可以利用回文的對稱特性，實作「中心擴展法 (Expand Around Center)」。藉由選擇一個字元（或兩個相鄰字元的間隙）作為中心點，由內而外同時向左與向右擴展，並檢查兩邊的字元是否相同，如此可以在 $O(n^2)$ 的時間複雜度內找出最長回文子字串。

## 2. 解題策略：雙指針由內而外擴展

我們可以針對字串中每一個可能的回文中心點進行擴展，找到並記錄最長的回文子字串的起始位置與長度。

1. **初始化**：
   - 設定 `start_index = 0` 紀錄最長回文子字串的起始索引，以及 `max_Len = 0` 紀錄其最大長度。
   - 字串總長度為 $n$。

2. **選擇中心點進行擴展**：
   - 使用迴圈讓索引 `i` 從 `0` 遍歷到 $n-1$。
   - 回文有兩種長度可能：
     - **長度為奇數**：以 `s[i]` 為中心（例如："aba"），我們將左右指標 `left` 與 `right` 都設為 `i` 進行擴展 `check(s, i, i)`。
     - **長度為偶數**：以 `s[i]` 與 `s[i+1]` 之間為中心（例如："abba"），我們將左右指標設為 `i` 與 `i+1` 進行擴展 `check(s, i, i+1)`。

3. **雙指標檢查與狀態更新 (`check` 函式)**：
   - 當左指標 `left >= 0`、右指標 `right < s.size()`，且 `s[left] == s[right]`（字元相同）時，表示目前子字串仍是回文，我們將 `left` 向左移 (`left--`)、`right` 向右移 (`right++`) 以繼續擴展。
   - 當擴展停止（迴圈結束）時，實際的回文範圍會是原本 `left + 1` 到 `right - 1` 的區間，因此我們將指標倒退一步：`left++` 與 `right--`。
   - 計算當前回文長度 `right - left + 1`。若大於目前記錄的 `max_Len`，則更新 `start_index = left` 與 `max_Len = right - left + 1`。

4. **回傳結果**：
   - 全部中心點都擴展並遍歷完畢後，利用 `s.substr(start_index, max_Len)` 擷取並回傳最長回文子字串。

## 3. 複雜度分析

- **時間複雜度**：$O(n^2)$。其中 $n$ 為字串 `s` 的長度。字串中有 $2n-1$ 個可能的回文中心（包含單一字元中心以及雙字元中心）。在擴展每一個中心點時，最差情況下（例如字串全為相同字元）需要花費 $O(n)$ 的時間橫跨整個字串，因此整體時間複雜度為 $O(n^2)$。
- **空間複雜度**：$O(1)$。我們僅使用了常數個額外變數如 `start_index`、`max_Len`、`left` 與 `right` 來儲存狀態與長度，沒有使用額外的陣列或資料結構，故空間複雜度為常數級別。
