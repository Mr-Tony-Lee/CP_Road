# LeetCode 3. Longest Substring Without Repeating Characters

## 1. 核心思維：滑動視窗 (Sliding Window)

我們的目標是找出給定字串 `s` 中，不含有重複字元的最長子字串的長度。

直觀的作法是檢查所有可能的子字串，但效率太低。我們可以利用「滑動視窗 (Sliding Window)」的技巧，透過維護一個不含重複字元的視窗 `[left, right]`，隨著 `right` 指標不斷向右掃描，動態調整 `left` 指標的位置，從而在一次遍歷中找出最長子字串。

## 2. 解題策略：陣列紀錄字元索引與動態調整左指標

我們可以利用一個長度為 128 的整數陣列 `char_index` 來紀錄每個字元最近一次出現的位置（索引值）。對於 ASCII 字元集，128 的長度已經足夠。

1. **初始化**：
   - 將 `char_index` 陣列初始化為 `-1`，表示字元尚未出現過。
   - 初始化左指標 `left = 0`，以及最長長度 `max_len = 0`。

2. **擴展右邊界 (`right` 指標)**：
   - 使用迴圈讓右指標 `right` 從 `0` 遍歷到字串尾端，取得當前字元 `cur_char = s[right]`。
   - 檢查當前字元是否在視窗內發生過重複：
     - 若 `char_index[cur_char] >= left`，代表 `cur_char` 已經存在於目前的滑動視窗 `[left, right]` 當中。
     - 此時我們需要縮小視窗，將左指標 `left` 更新為重複字元上次出現位置的下一個位置，即 `left = char_index[cur_char] + 1`，從而將重複字元排除在視窗外。

3. **更新狀態與最長長度**：
   - 更新 `char_index[cur_char] = right`，紀錄當前字元最新出現的位置。
   - 計算當前無重複字串的長度 `right - left + 1`，並更新最長長度 `max_len = max(max_len, right - left + 1)`。

當字串掃描完畢，`max_len` 即為我們要求的最長子字串長度。

## 3. 複雜度分析

- **時間複雜度**：$O(n)$。其中 $n$ 為字串 `s` 的長度。右指標 `right` 會走訪字串一次，而左指標 `left` 直接透過陣列紀錄的索引值進行跳躍更新。整體遍歷所需時間正比於字串長度。
- **空間複雜度**：$O(m)$。其中 $m$ 為字元集大小。由於我們使用的是 ASCII 字元，字元集大小固定為 128，建立了一個長度為 128 的 `vector<int>` 來紀錄出現位置，因此空間複雜度為常數級別，可看作 $O(1)$。
