# LeetCode 7. Reverse Integer

## 1. 核心思維：逐位反轉與溢出檢查 (Digit Reversal and Overflow Check)

我們的目標是將給定的 32 位元有號整數 `x` 的每一位數字反轉。

直觀的作法是將整數轉為字串再進行反轉，但這會使用到額外的空間，且未遵循環境可能無法儲存 64 位元整數的嚴格限制。較佳的作法是直接以數學運算「取餘數」與「除法」來逐位取出數字並重新組合，同時必須在每次放大並累加結果前，嚴格檢查是否會發生整數溢出 (Integer Overflow) 問題。

## 2. 解題策略：取餘數與提早判斷溢出

我們可以不斷從原數字 `x` 提取出最後一位數字，並將其附加到反轉後的數字 `rev` 的末端。

1. **初始化**：
   - 宣告一個變數 `rev = 0` 用來儲存反轉後的結果。

2. **逐位反轉**：
   - 使用 `while` 迴圈，只要 `x != 0` 就繼續執行。
   - 每次從原數字 `x` 提取最後一位數字：`int pop = x % 10`。
   - 將原數字 `x` 捨棄最後一位數字：`x /= 10`。

3. **溢出檢查 (Overflow Check)**：
   - 由於題目限制環境僅能儲存 32 位元有號整數 `[-2^31, 2^31 - 1]` (即 `[-2147483648, 2147483647]`)。在將 `pop` 加到 `rev` 之前（即執行 `rev = rev * 10 + pop`），必須先確認 `rev * 10` 會不會導致溢位。
   - **正數溢出檢查**：
     - 若 `rev > INT_MAX / 10`，則 `rev * 10` 必定會超過 `INT_MAX`。
     - 若 `rev == INT_MAX / 10`，則只要 `pop > 7` （因為 `INT_MAX` 的個位數是 7），`rev * 10 + pop` 就會溢出。
     - 遇到上述情況直接回傳 `0`。
   - **負數溢出檢查**：
     - 若 `rev < INT_MIN / 10`，則 `rev * 10` 必定會低於 `INT_MIN`。
     - 若 `rev == INT_MIN / 10`，則只要 `pop < -8` （因為 `INT_MIN` 的個位數是 8，負數為 -8），`rev * 10 + pop` 就會溢出。
     - 遇到上述情況直接回傳 `0`。

4. **更新反轉結果**：
   - 確認安全後，將反轉結果更新為 `rev = rev * 10 + pop`。

5. **回傳結果**：
   - 迴圈結束後，回傳最終的反轉結果 `rev`。

## 3. 複雜度分析

- **時間複雜度**：$O(\log_{10} x)$ 或 $O(1)$。操作的次數與整數 `x` 的十進位位數成正比，即大約 $\log_{10} x$ 次。因為題目限制 `x` 為 32 位元整數，其位數最多只有 10 位，因此時間複雜度也可以視為常數級別 $O(1)$。
- **空間複雜度**：$O(1)$。我們僅使用了 `rev` 與 `pop` 等幾個區域變數來記錄狀態與數字，不論輸入多大，額外使用的記憶體空間皆為常數大小，故空間複雜度為常數級別。
